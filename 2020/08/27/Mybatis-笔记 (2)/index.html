    4、输入类型parameterType接受标识符#{}、${}的区别

    4.1简单类型

    ①接受方式：

    方式一：#{任意值}
    方式二：${value} ，其中的标识符只能是value

    或

    ②两种方式区别

    #{}自动给String类型加上’’ （自动类型转换）

    ${} 原样输出，但是适合于动态排序（动态字段），手动添加

    '${value}'

    动态排序：
    select stuno,stuname,stuage from student order by ${value} asc

    ③此外：

    #{}可以防止SQL注入
    ${}不防止

    ④${}、#{}相同之处：
    都可以 获取对象的值 （嵌套类型对象）

    4.2引用类型

    方式一：#{属性名}
    方式二：${属性名}

    五、查询缓存和逆行工程

    1、查询缓存

    1、一级缓存：同一个SqlSession对象

    MyBatis默认开启一级缓存，如果用同样的SqlSession对象查询相同的数据，则只会在第一次查询时向数据库发送SQL语句，并将查询的结果放入到SqlSession对象中（作为缓存存在）；后续再次查询该同样的对象时，则直接从缓存中查询该对象即可（即省略了数据库的访问）

    两次查询只有一天查询语句：

    增删改的commit会清理缓存：第二次查询时又连接了数据库：

    2、二级缓存

    1、自带二级缓存：同一个namespace生成的一个或多个mapper对象

    ①namespace：接口的全类名（包名.类名），通过接口可以产生代理对象（studentMapper对象）

    ②开启二级缓存：conf.xml和XXXMapper.xml

    ③序列化缓存对象以及缓存对象的级联属性、父类
    根据异常提示：NotSerializableException可知，MyBatis的二级缓存是将对象放入硬盘文件中
    序列化：内存->硬盘
    反序列化：硬盘->内存
    ④触发将对象写入二级缓存的时机：SqlSession对象的close()方法。

    ⑤测试：

    ⑥禁用二级缓存：useCache=”false”

    ⑦清理缓存：

    方式一：增删改的commit（防止脏数据）

    方式二：flushCache=”true”

    2、第三方缓存：
    要想整合三方提供的二级缓存 （或者自定义二级缓存），必须实现org.apache.ibatis.cache.Cache接口，该接口的默认实现类是PerpetualCache

        整合ehcache二级缓存：
    a.
    ehcache-core.jar
    mybatis-Ehcache.jar
    slf4j-api.jar

    b.编写ehcache配置文件 Ehcache.xml

    c.开启EhCache二级缓存

        在xxxMapper.xml中开启
        <cache  type="org.mybatis.caches.ehcache.EhcacheCache">
            <!-- 通过property覆盖Ehcache.xml中的值 -->
            <property name="maxElementsInMemory" value="2000"/>
            <property name="maxElementsOnDisk" value="3000"/>
        </cache>
