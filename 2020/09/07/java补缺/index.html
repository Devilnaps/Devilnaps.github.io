3、对象的属性：就是全局变量，

全局变量：方法以外，类以内； 有初始值（数据类型的默认值 int 0）
局部变量：方法内部 ;没有初始值，在使用前 必须赋初值

局部变量可以和全局变量重名，并且局部变量优先级高

4、方法的调用的常见情况：

同一个类中：都有staitc，或都没static，或 没的调有的:直接调用
有static 不能直接调用 没有static的方法

不在同一个类中：
万能方法： new 对象，对象.方法() ;
Person2 p = new Person2();
p.eatFruit();
调用的另一个方法必须是静态的：Person2.eatFruit(); //类名.方法() 其中的方法必须是static修饰的

static方法 是 类级别的，属于类
非static方法 是 对象级别，属于对象

对象.static方法() 对象.非static方法()
类.static方法()
类有的，对象自然有；
对象有的，类不一定有

10、构造方法

含参构造方法的作用：

1.实例化（产生对象）

2.一次性给多个属性赋值（set方法只能一次一个）

3、多个构造方法之间 不能循环调用

4、this 关键字可使本类构造器之间相互调用，但必须放置在构造器第一行

5、子类继承父类： private 、构造方法是不能被继承（不能被重写）的。但是子类可以显示的调用父类构造（super）

6、this 和super在一个构造器中只能二选一，且都要放在第一行

13、 final

final修饰的类 ：不能被继承
final修饰的方法：不能被重写
final修饰的变量、属性：不能被修改

final修饰的引用：引用不能被改变，但是引用的对象的值可以改变

23、异常（try、catch、finally、throw、throws）

1、try…catch：（可以使用if…else 替代）

2、异常的基类为Throwable

Throwable子类分为Error（错误，程序员不能处理）和Exception类（一般讨论的异常，需要处理的异常）

Exception类分为RunTimeException（运行时异常，在编辑时不报错）和CheckException（非运行时异常、检查异常，在编写时提示）

但是，CheckException不继承Exception类，只是一个统称

3、try…catch…finally

自己不能处理的异常，通过throws提交给上级方法处理

如果在try中发生了异常，则try后续代码不执行，直接跳到catch执行

finally:最终都要执行的内容（即使遇到return，依然执行，除非关闭虚拟机：System.exit(1);）

4、多个catch捕获不同的异常：

只执行一个

先catch范围小的，后catch范围大的

5、throws：把当类的异常抛给调用该类的上级解决

6、throw：自定义异常时使用

①继承Exception或Throwable或其他异常（如NullPointerException等）

②声明一个异常并且通过try…catch 或throws处理异常

26、网络编程

1、一对一

ip：网络唯一标识符，由32位二进制组成（4个8位二进制，每个最大255）

协议：网络通讯的规则、标准、约定

TCP：面向连接的、可靠的（不丢失、不重复、有序）、基于字节流的传输通讯协议，传输速度慢

UDP：无连接的协议，在传输数据之前，客户端和服务端之间不建立和维护连接，提供不可靠的数据传输

socket：套接字，基于TCP协议的网络通讯，提供双向安全连接的网络通信。socket需要借助于数据流完成数据的传输。

实现：

  1. 建立连接
  2. 通过socket生成InputStream和OutputStream（准备发送数据）
  3. 使用InputStream和OutputStream收发数据
  4. 关闭数据流
